# 集控总览 - 重建设计文档

## 概述

将现有通用型仪表盘（CentralOverview.tsx）改造为以**设备监控为核心**的数据总览页面，提供设备盘点、达标率监控、阈值自定义、设备详情钻取等功能。

## 范围

- 仅重写 `src/app/components/CentralOverview.tsx`（约 470 行 → 预计 800-1000 行）
- 集控管理下其他子页面（教室管理、设备管理、实时流、信息发布、安全策略、运行日志）不受影响
- `permissions.ts` 中 `central-overview` 页面的按钮配置不变（当前无按钮）

## 页面布局（方案 A）

页面从上到下分为三大区域：

```
┌──────────────────────────────────────────────────────────┐
│  📊 概览卡片区（6 个卡片，每行 3 个，自适应）               │
│  监管设备数 | 设备使用率 | 网络达标率                       │
│  硬件达标率 | 流畅度达标率 | 安全达标率                     │
├──────────────────────────────────────────────────────────┤
│  ⚙️ 阈值设置  📥 导出数据  [时间维度切换]                  │
├──────────────────────────────────────────────────────────┤
│  设备列表区                                              │
│  [搜索框] [状态筛选] [达标筛选]                           │
│  ┌────────────────────────────────────────────────┐      │
│  │ 表格：设备名称 | 编号 | 位置 | 状态 | ... | 操作│      │
│  │ 点击"查看详情" → 弹出详情弹窗（5 个 Tab）        │      │
│  └────────────────────────────────────────────────┘      │
│  分页                                                   │
└──────────────────────────────────────────────────────────┘
```

## 1. 仿真数据

### 设备数据（30-50 条）

覆盖多教学楼、多楼层，混合在线/离线/异常状态，混合达标/不达标情况。

**教学楼/楼层结构：**
- 东教学楼：一楼（101-103）、二楼（201-202）、三楼（301）
- 西教学楼：一楼（101-102）、二楼（201-203）
- 综合楼：一楼（101-104）、二楼（201-202）

**状态分布：** 约 80% 在线、10% 离线、10% 异常

**达标分布：** 各达标率指标独立随机生成，部分设备部分指标不达标

### 硬件参数（≥13 项）

| 参数项 | 数据类型 | 达标条件 |
|-------|---------|---------|
| CPU 型号 | 字符串 | - |
| CPU 使用率 | 百分比 | ≤ 80% |
| CPU 平均温度 | 摄氏温度 | ≤ 75°C |
| 内存大小 | GB | ≥ 4 GB |
| 内存使用率 | 百分比 | ≤ 80% |
| 系统盘容量 | GB | - |
| 系统盘使用率 | 百分比 | ≤ 85% |
| 数据盘容量 | GB | - |
| 数据盘使用率 | 百分比 | ≤ 85% |
| 操作系统 | 字符串 | - |
| 屏幕分辨率 | 字符串 | - |
| 触摸屏状态 | 布尔 | - |
| 音响状态 | 布尔 | - |
| 麦克风状态 | 布尔 | - |
| 摄像头状态 | 布尔 | - |

### 达标率聚合

6 个概览卡片的聚合数据从设备列表计算得出：
- **监管设备数：** 设备总数
- **设备使用率：** 最近 7 天有活跃记录设备数 / 总设备数
- **网络达标率：** 网络达标设备数 / 总设备数
- **硬件达标率：** 硬件全部达标设备数 / 总设备数
- **流畅度达标率：** 流畅度达标设备数 / 总设备数
- **安全达标率：** 安全达标设备数 / 总设备数

## 2. 概览卡片

### 展示样式

- 每个卡片：图标 + 标题 + 大号百分比/数值
- 达标率卡片右上角显示分子/分母（如 `276/328`）
- 颜色规则（基于自定义阈值 T）：
  - ✅ 绿色：≥ T
  - ⚠️ 黄色：≥ T × 90% 但 < T（临界区）
  - ❌ 红色：< T × 90%
- 设备使用率：绿色表示使用充分，黄色表示偏低，红色表示严重偏低

### 布局

- 桌面端：每行 3 个，共 2 行
- 平板端：每行 2 个，共 3 行
- 移动端：每行 1 个，共 6 行

## 3. 阈值设置

### 入口

概览卡片右下侧操作栏的「阈值设置」按钮，紧挨「导出数据」

### 弹窗内容

| 指标 | 默认值 | 输入范围 |
|------|-------|---------|
| 设备使用率 | 60% | 0-100% |
| 网络达标率 | 90% | 0-100% |
| 硬件达标率 | 90% | 0-100% |
| 流畅度达标率 | 85% | 0-100% |
| 安全达标率 | 95% | 0-100% |

### 交互

- 每个指标带 +/− 微调按钮及直接输入
- 底部「恢复默认」按钮
- 保存后写入 localStorage，卡片颜色立即更新
- 取消不保存

## 4. 导出数据

- 点击「导出数据」按钮
- 导出为 CSV 格式（也可考虑使用 SheetJS 导出为 xlsx）
- 内容：设备列表全部列，包含每个设备的达标状态
- 使用浏览器的下载功能（创建 Blob + URL.createObjectURL + 模拟点击）

## 6. 设备详情弹窗

点击「查看详情」弹出大型 Dialog，5 个 Tab 切换：

### Tab 1 — 设备信息

顶部展示设备基础信息卡片（名称、编号、状态、位置、最近活跃），下方展示网络详情：

| 项 | 值 |
|----|-----|
| 上行网速 | XX Mbps |
| 下行网速 | XX Mbps |
| 流入流量 | XX MB/GB |
| 流出流量 | XX MB/GB |

### Tab 2 — 硬件参数

≥13 项硬件参数以两列卡片列表展示，每项带达标标识（✅/❌）。

分组：
- **计算：** CPU 型号、CPU 使用率、CPU 温度
- **存储：** 内存大小、内存使用率、系统盘、系统盘使用率、数据盘、数据盘使用率
- **系统：** 操作系统、屏幕分辨率
- **外设：** 触摸屏、音响、麦克风、摄像头

### Tab 3 — 安全情况

安全检查项表格：系统补丁、杀毒软件、防火墙、违规软件、USB 管控
底部展示整体安全达标结论（✅/❌）

### Tab 4 — 使用分析

- **设备使用时间段分布：** 柱状图（0-24 时各时段活跃次数）
- **软件使用时长排行：** 横向条形排行（Top 5）
- **软件使用列表：** 表格（软件名、使用时长、打开次数）
- **老师使用情况：** 最近使用的老师姓名、使用时间段

### Tab 5 — 异常记录

表格：时间 | 异常类型 | 描述 | 状态（已恢复/未恢复/处理中）
异常类型包括：网络断开、CPU 过热、内存不足、磁盘空间不足、系统崩溃、外设异常等

## 7. 技术实现

### 状态管理

- 所有数据为前端仿真 mock 数据，定义在组件内部或独立的数据文件
- 阈值设置存储于 `localStorage`（key: `central-overview-thresholds`）
- 使用 `useMemo` 聚合概览卡片数据

### 组件结构

```

├── StatCard               # 概览卡片（可复用）
├── ThresholdDialog        # 阈值设置弹窗
├── DeviceTable            # 设备列表表格
├── DeviceDetailDialog     # 设备详情弹窗
│   ├── DeviceInfoTab      # Tab 1
│   ├── HardwareTab        # Tab 2
│   ├── SecurityTab        # Tab 3
│   ├── UsageTab           # Tab 4（含迷你图表）
│   └── AnomalyTab         # Tab 5
```

### 依赖

已有依赖即可覆盖：
- `@mui/material` — 对话框、表格、卡片、Tabs、按钮等
- `@mui/icons-material` — 图标
- `recharts` — Tab 4 中的时段分布柱状图
- `tailwindcss` — 样式

### 导出 CSV

原生实现，无需额外库：
```typescript
function exportToCSV(data: Device[], filename: string) {
  const headers = [...];
  const rows = data.map(d => [...]);
  const csv = [headers, ...rows].map(r => r.join(',')).join('\n');
  const blob = new Blob(['﻿' + csv], { type: 'text/csv;charset=utf-8;' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url; a.download = filename;
  a.click(); URL.revokeObjectURL(url);
}
```

## 8. 删除的旧功能

以下 CentralOverview.tsx 中的功能将被移除：
- 设备活跃分布饼图（设备活跃等级分布）
- 开机长时间未使用统计柱图
- 网站访问情况图表
- 所有与设备无关的通用统计

这些功能与新的以设备为核心的需求不匹配。

# 设备管理页面设计文档

## 概述

完善集控模块下的设备管理页面，实现对全校教室终端设备的集中管理、远程监控和指令下发。

## 页面结构

```
┌─────────────────────────────────────────────────────────┐
│  设备管理 （标题 + 副标题）                               │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  统计卡片行（总设备 / 在线 / 离线 / 异常）             │ │
│  └─────────────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  搜索筛选栏 + 视图切换                               │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                          │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  （表格视图 或 卡片网格视图）                           │ │
│  │  设备列表区域                                        │ │
│  │  带分页控件                                          │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                          │
│  （选中设备时浮现批量操作栏）                               │
└─────────────────────────────────────────────────────────┘
```

## 1. 顶部统计卡片行

显示 4 个统计卡片，复用 CentralOverview 中的 StatCard 风格：

| 卡片 | 图标 | 数据来源 |
|------|------|---------|
| 总设备数 | Devices | 设备列表总长度 |
| 在线 | CheckCircle（绿色） | status === 'online' |
| 离线 | Cancel（灰色） | status === 'offline' |
| 异常 | Warning（红色） | status === 'abnormal' |

## 2. 搜索与筛选

### 搜索
- 支持设备名称、设备编号、IP 地址模糊搜索（输入即搜，无搜索按钮）
- 搜索框左侧有放大镜图标

### 筛选维度

| 筛选 | 控件 | 选项 | 说明 |
|------|------|------|------|
| 状态 | Select 下拉 | 全部 / 在线 / 离线 / 异常 | |
| 楼栋 | Select 下拉 | 全部 + 动态从设备数据聚合 | 与楼层联动 |
| 楼层 | Select 下拉 | 全部 + 根据选中楼栋过滤楼层 | |
| 操作系统 | Select 下拉 | 全部 / Windows 10 / Windows 11 / HarmonyOS | |
| 在线时长 | Select 下拉 | 全部 / ＜1小时 / 1-6小时 / ＞6小时 | 基于 lastActive 计算 |

### 视图切换
- 筛选栏右侧放置「列表视图」(TableView) 和「网格视图」(GridView) 两个图标按钮
- 选中状态高亮，切换时保持当前筛选条件和分页

### 重置
- 「重置」文本按钮，一键清除所有搜索和筛选条件

## 3. 表格视图

### 列定义

| 列 | 宽度 | 内容 | 说明 |
|---|------|------|------|
| 勾选 | 40px | Checkbox | 顶部全选 |
| 缩略图 | 120px | 模拟桌面截图 | 在线显示彩色缩略图，离线显示灰色占位 |
| 设备名称/编号 | 200px | 名称 + 灰色编号 | 离线时编号位置显示下线原因 |
| 所属位置 | 150px | 楼栋/楼层/教室 | |
| 状态 | 80px | Chip | 在线绿/离线灰/异常红 |
| 系统/资源 | 130px | OS + CPU/内存进度条 | 仅在线设备显示资源占用 |
| 操作 | 90px | 「操作」下拉按钮 | 弹出远程指令菜单 |

### 缩略图交互
- 鼠标悬停出现「点击查看」提示
- 点击弹出远程桌面对话框
- 在线设备：模拟桌面截图（蓝色/绿色渐变背景 + 图标占位）
- 离线设备：灰色背景 + 电源线断开图标

### 分页
- 每页 15 条
- 显示总数：「共 N 台设备」
- 底部分页控件：上一页 / 页码 / 下一页 / 每页条数选择

## 4. 卡片网格视图

### 卡片布局
- 每行 3-4 张卡片（响应式）
- 卡片上方有全选 Checkbox

### 卡片内容（从上到下）
1. 大缩略图区域（约为表格缩略图的 3 倍）
2. 缩略图右下角：状态小圆点（绿/灰/红）
3. 设备名称（粗体）
4. 状态 Chip + 系统版本
5. CPU 使用率进度条（仅在线设备显示）
6. 「操作」按钮

### 交互
- 鼠标悬停 → 缩略图上浮现快捷操作图标（截屏、锁屏、远程桌面）
- 点击卡片 → 进入远程桌面对话框
- 响应式断点：≥1200px 4列 / ≥768px 3列 / <768px 2列

| 重启 | 远程重启系统 | ✅ | ✅ |

| 远程设置 | 修改设备配置 | ✅ | ❌ |
| 系统更新 | 触发系统更新 | ✅ | ✅ |

### 操作入口

**单设备操作：** 表格行「操作」下拉菜单 / 卡片底部「操作」按钮
- 菜单按大类分组（系统控制 ▸ 屏幕控制 ▸ 等）
- 子菜单 hover 展开

**批量操作：** 勾选 ≥ 2 台设备后，在表格/卡片上方（或底部固定）浮现批量操作栏
- 显示「已选中 N 台设备」
- 常用指令显示为按钮（关机 / 重启 / 锁屏 / 打铃 / 截屏 / 消息）
- 更多指令通过「文件分发▸」「定时任务▸」等分组按钮
- 「取消选择」按钮清除所有勾选

### 执行反馈

指令执行结果以弹出式通知列表反馈：
```
正在执行：关机
✅ 教学楼101  已完成
✅ 教学楼102  已完成
❌ 教学楼103  设备离线，发送失败
⏳ 教学楼104  等待中...

完成 2 / 失败 1 / 等待中 1  [我知道了]
```

## 6. 远程桌面对话框

### 布局
- 左侧约 75%：大屏画面区域
- 右侧约 25%：设备信息 + 快捷指令侧边栏
- 底部：操作记录栏

### 功能
- 模拟桌面截图显示
- 最后刷新时间显示
- 自动刷新开关（可开/关定时刷新）
- 右侧快捷指令面板（锁屏 / 截屏 / 关机 / 重启 / 打铃 / 消息）
- 全屏模式
- 底部操作时间线记录

## 7. 文件分发弹窗

### 字段
- 目标设备列表（只读显示）
- 文件选择区：点击选择 + 拖拽上传
- 已选文件列表：文件名、大小、移除按钮
- 目标路径：输入框 + 浏览按钮（C:\Users\Public\Documents\教学资源\）
- 选项：接收后自动打开 / 覆盖已有文件

### 传输过程
- 开始传输后显示每台设备的进度条
- 列示：设备名、进度百分比、传输速度
- 完成状态标记 ✅

## 8. Mock 数据策略

- 复用 CentralOverview 中 BUILDINGS → floors → rooms 结构生成的 17 台设备数据
- 扩展每个设备对象，增加：
  - `ip: string` — IP 地址
  - `os: string` — 操作系统版本
  - `screenUrl: string` — 模拟桌面缩略图（用渐变+图标占位）
  - `onlineDuration: number` — 在线时长（分钟）
  - `volume: number` — 当前音量 0-100
- 指令执行结果用 setTimeout 模拟 1-3 秒后随机成功/失败

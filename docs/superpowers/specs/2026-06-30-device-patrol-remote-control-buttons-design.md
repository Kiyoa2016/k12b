# 设备巡视弹窗 — 远程控制指令按钮设计

## 概述

在设备巡视（DevicePatrol）的视频巡视弹窗（PatrolDialog）中，增加远程控制指令按钮列表。用户可在查看教室实时画面时，直接对目标设备执行开机、关机、远程喊话、发送信息等操作。

## 改动范围

仅涉及一个文件：`src/app/components/DevicePatrol.tsx` 中的 `PatrolDialog` 组件。

## 布局

当前弹窗内容区为左右两栏：

```
┌─────────────────────────────────────┬──────────────┐
│                                     │  视频源列表    │
│          视频流画面                  │  ○ 摄像头 1   │
│                                     │  ● 屏幕录制 1  │
│                                     │  ○ 摄像头 2   │
│                                     │              │
├─────────────────────────────────────┤              │
│  🔌开机  ⏻关机  📢远程喊话  ✉️发送信息 │              │
└─────────────────────────────────────┴──────────────┘
```

- 左侧视频区域下方增加一行操作按钮栏，高度约 48px，内边距上下 8px
- 按钮与视频区域之间用分隔线（`Divider` 或浅色边框）视觉区隔
- 右侧频道列表结构不变

## 按钮定义

| 按钮 | 图标 (MUI) | 文字颜色 | 点击提示 |
|------|-----------|---------|---------|
| 开机 | `PowerSettingsNew` | 绿色 `#16a34a` | "已发送开机指令至 {教室名}" |
| 关机 | `PowerOff` | 红色 `#ef4444` | "已发送关机指令至 {教室名}" |
| 远程喊话 | `Campaign` | 蓝色 `#3b82f6` | "已向 {教室名} 发起远程喊话" |
| 发送信息 | `Send` | 蓝色 `#3b82f6` | "已发送信息至 {教室名}" |

- 按钮使用 MUI `<Button>` 组件，`variant="text"`，前后各带图标
- 按钮之间间距 `gap: 16px`，水平居中或左对齐排列

## 交互行为

1. **正常状态**：按钮可点击，点击后弹出 `Snackbar`（MUI Alert 风格）提示操作已发送
2. **空状态**：当无可用的视频源（`channels` 为空或 `current === null`）时，按钮显示但处于 `disabled` 状态，hover 时 tooltip 提示"当前无视频连接"
3. **预留给未来**：每个按钮的 `onClick` 处理函数中预留 `// TODO: 调用实际 API` 注释，方便后续对接真实后端接口

## 状态管理

- 无需新增全局状态
- Snackbar 使用 `useState` 局部管理 `{ open, message }`，自动 3 秒后关闭

## 未涉及

- 真实 API 调用（仅模拟提示）

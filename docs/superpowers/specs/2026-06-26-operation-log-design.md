在"集控管理"下完善"运行日志"功能，展示平台下发的操作指令（平台审计日志）及指令在终端上的执行结果（终端审计日志）。

## 布局方案

上下结构，顶部标题区，中间 Tab 切换，下方统一筛选栏 + 表格。

```
┌──────────────────────────────────────────────────────────────┐
│  📋 运行日志                          [导出日志] [清除日志]    │
│  查看系统运行日志，包含平台下发的操作指令及终端执行情况         │
├──────────────────────────────────────────────────────────────┤
│  [平台审计日志]  [终端审计日志]                                │
├──────────────────────────────────────────────────────────────┤
│  筛选栏                                                      │
│  ┌────────────────────────────────────────────────────┐       │
│  │  表格区域                                            │       │
│  └────────────────────────────────────────────────────┘       │
│  分页                                                         │
└──────────────────────────────────────────────────────────────┘
```

两个 Tab 共用工具栏（导出、清除），但筛选条件和表格列各自独立。

## Tab 1：平台审计日志

记录管理人员或系统自动下发的操作指令。

### 表格列定义

| 列 | 说明 | 宽度 |
|----|------|------|
| 序号 | 自动编号 | 50px |
| 时间 | 操作时间，格式 YYYY-MM-DD HH:mm | 140px |
| 操作人员 | 管理员账号，或"系统自动" | 100px |
| 操作类型 | 指令分类，显示为 Chip | 100px |
| 操作内容 | 具体指令描述 | 200px+ (flex) |
| 目标设备 | 点击可展开查看具体设备列表 | 80px |
| 状态 | 指令下发状态 | 80px |
| 操作 | 查看详情 | 80px |

### 操作类型（Mock 数据）

设备重启、磁盘清理、文件迁移、磁盘格式化、信息发布、安全策略变更、固件升级、系统配置修改

### 状态定义

- **已下发** — 指令已成功下发到所有目标设备
- **全部成功** — 所有目标设备执行成功
- **部分成功** — 部分设备执行成功（显示如 "3/5 台成功"）
- **失败** — 指令下发失败或全部执行失败

### 目标设备点击展开

点击"目标设备"列的数字（如 "3 台"），弹出 Dialog 显示具体设备列表：
- 设备名称、设备编号、所属位置、执行状态、执行时间、详情/失败原因

### 筛选条件

- 时间范围：开始日期 ~ 结束日期（datetime-local picker）
- 操作类型：下拉多选（全部 / 各操作类型）
- 操作人员：文本搜索
- 状态：下拉筛选（全部 / 已下发 / 全部成功 / 部分成功 / 失败）

## Tab 2：终端审计日志

记录各终端收到指令后的执行结果。

### 表格列定义

| 列 | 说明 | 宽度 |
|----|------|------|
| 序号 | 自动编号 | 50px |
| 时间 | 终端执行/上报时间 | 140px |
| 设备名称 | 终端设备名称 | 150px |
| 设备编号 | 设备编码 | 110px |
| 所属位置 | 教学楼 / 楼层 / 教室 | 180px |
| 指令类型 | 对应操作类型 Chip | 100px |
| 指令内容 | 具体指令描述 | 200px+ (flex) |
| 执行状态 | 成功/失败/执行中 | 80px |
| 详情 | 失败原因或补充信息 | 120px |

### 执行状态定义

- **成功** — 终端上报执行成功
- **失败** — 终端上报执行失败，含原因
- **执行中** — 指令已下发但终端尚未上报结果

### 筛选条件

- 时间范围：开始日期 ~ 结束日期
- 设备名称/编号：文本搜索
- 指令类型：下拉筛选
- 执行状态：下拉筛选（全部 / 成功 / 失败 / 执行中）

## 工具栏

两个 Tab 共享页面顶部的操作按钮：

- **导出日志** — 根据当前 Tab 导出 CSV（导出平台日志或终端日志）
- **清除日志** — 弹出二次确认 Dialog，确认后清除当前 Tab 对应数据

## 数据设计

### Mock 生成策略

参考 CentralOverview / NewsBroadcast 等模块的 mock 方式：

- 平台日志：从预定义的操作类型、操作人列表中随机组合生成 20-30 条记录，时间为最近 30 天内
- 终端日志：为每条平台日志中每台目标设备生成一条执行记录（成功概率约 80%，失败约 15%，执行中约 5%）
- 终端设备名复用其他模块的 DEVICE_NAMES 列表

### 状态联动

平台日志的"部分成功"状态取决于其关联的终端日志中成功比例：
- 全部终端成功 → 平台日志状态 = "全部成功"
- 部分终端成功 → "部分成功"（显示 x/y 台成功）
- 全部失败 → "失败"
- 刚下发未反馈 → "已下发"

## 1. 产品概述

### 1.1 定位

语音控制系统是后台管理系统的**配置管理模块**，面向教室场景，为教室桌面端提供语音唤醒 + 语音指令控制的能力。管理端负责配置下发，桌面端负责指令执行。

### 1.2 架构概览

```
┌─────────────────────────────────────────────────┐
│              后台管理系统（配置端）                │
│                                                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │ 基础配置  │ │ 指令管理  │ │ 词库管理  │          │
│  └──────────┘ └──────────┘ └──────────┘          │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │ 权限控制  │ │ 日志统计  │ │ 语音管理  │          │
│  │          │ │          │ │ 菜单页   │          │
│  └──────────┘ └──────────┘ └──────────┘          │
```

## 2. 管理端模块设计

### 2.1 基础配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 唤醒词 | "果仁果仁" | 支持自定义，2-6个中文字，支持试听 |
| 唤醒灵敏度 | 中 | 高/中/低 三档 |
| 语音引擎 | 默认固定 | 固定使用选定引擎，不提供切换 |
| TTS 音色 | 女声（亲切） | 支持试听，多音色可选 |
| TTS 语速 | 正常 | 三段：偏慢 / 正常 / 偏快 |
| TTS 音量 | 70% | 滑块 0-100%，独立于系统音量 |
| 语言 | 普通话 | 依据学校需要扩展 |
| 降噪模式 | 普通 | 低噪（办公室）/ 普通（教室）/ 高噪（操场活动） |
| 唤醒后超时 | 8 秒 | 唤醒后无有效指令自动回到待唤醒 |

### 2.2 指令管理

#### 2.2.1 指令分类

| 类别 | 指令示例 | 参数 | 说明 |
|------|---------|------|------|
| **课件控制** | 下一页 / 上一页 / 翻到第X页 / 回到首页 / 最后一页 | 页码：数字 | 控制当前打开的课件 |
| | 打开《XXX》课件 / 关闭课件 | 课件名：模糊匹配课件库 | 支持模糊搜索 |
| **板书控制** | 画笔 / 红色笔 / 橡皮 / 撤销 / 重做 | 颜色：预定义颜色词表 | 不持支持自定义颜色名 |
| | 清屏 / 保存板书 / 新建页面 | — | 清屏前需确认 |
| **教学工具** | 开始计时X分钟 / 倒计时X秒 / 停止计时 | 时间：1-60 | — |
| | 打开聚光灯 / 关闭聚光灯 / 打开放大镜 / 打开骰子 / 打开小黑板 | — | — |
| **多媒体控制** | 播放 / 暂停 / 快进30秒 / 快退30秒 / 重播 | 秒数：固定值 | 控制当前已打开的音视频 |
| | 音量调大 / 音量调小 / 静音 / 音量调到X% | 音量：20%/40%/60%/80%/100% | 按固定档位调节 |
| **课堂互动** | 随机点一名同学 / 再来一个 | — | 从当前班级花名册抽取 |
| | 第一组加1分 / 第三组扣1分 | 组别：数字 分数：数字 | 同步到小组积分榜 |
| **系统操作** | 打开 [目标] | 目标：应用/文件/网页/功能 | 灵活参数匹配，不穷举 |
| **语音搜索** | 搜索XXX / 找关于XXX的课件 | 关键词：任意文本 | 搜索校本资源库+云端 |
| | XXX是YYY / 什么是XXX | 任意知识点 | AI返回简短答案弹窗 |

#### 2.2.2 "打开"指令的灵活参数设计

"打开 [目标]" 不做穷举匹配，采用分类分发机制：

```
用户说: "打开浏览器"
→ NLU 解析出: intent="打开", target="浏览器"
→ 分发到 "系统应用匹配器"
→ 匹配到系统已注册的浏览器 → 执行

用户说: "打开勾股定理课件"
→ NLU 解析出: intent="打开", target="勾股定理课件"
→ 分发到 "课件匹配器"
→ 在课件库中模糊搜索 → 匹配到 → 执行
```

匹配顺序：
1. **系统应用** — 扫描系统已安装的应用列表
2. **课件/文件** — 匹配资源库中的文件名
3. **网页快捷方式** — 匹配预置的 URL 快捷方式
4. **系统功能** — 控制面板、设置等系统功能

#### 2.2.3 管理端指令管理功能

| 功能 | 说明 |
|------|------|
| 指令清单 | 按类别展示所有可用指令，支持搜索 |
| 类别管理 | 类别支持增删，删除类别后该类别下的指令自动归入"未分类" |
| 启用/禁用 | 按指令类别或逐条开关 |
| 自定义指令 | 用户将短语映射到操作（"开始上课"→ 多个操作序列） |
| 冲突检测 | 自定义指令与系统指令冲突时提示 |
| 新增指令 | 支持自定义新增指令，指定归属类别 |

### 2.3 词库管理

#### 2.3.1 词库分类

| 词库类型 | 内容 | 维护方式 |
|---------|------|---------|
| 学科术语 | 勾股定理、光合作用、二次函数…… | 预置（按学科） + 自定义 |
| 教材词库 | 人教版/北师大版 各章节名称 | 预置 |
| 自定义词库 | 教师姓名、班级名称、校本课程 | 管理员手动添加 |
| 课件索引 | 资源库中课件/文件名称 | 自动同步 |
| 热词 | 近期高频词汇，动态提升权重 | 系统自动学习 |

#### 2.3.2 词库管理功能

| 功能 | 说明 |
|------|------|
| 词条增删改 | 添加/编辑/删除自定义词条 |
| 批量导入 | 支持 CSV/TXT 批量导入 |
| 词库生效范围 | 全局 / 指定班级 / 指定教师 |
| 自动同步 | 课件库变更时自动更新课件索引 |
| 纠偏建议 | 根据识别失败日志推荐新词条 |

### 2.4 权限控制

语音权限集成到现有**角色管理系统**中，在"编辑权限"弹窗中新增"语音指令"页面 Tab：

| 权限项 | 粒度 |
|-------|------|
| 语音功能总开关 | 角色级别 |
| 指令类别权限 | 按类别开关（课件控制/板书/教学工具/多媒体/课堂互动/系统操作/语音搜索） |
| TTS 播报开关 | 角色级别 |
| 敏感操作需确认 | 清屏/删除/关机等操作需要二次语音确认 |

### 2.5 日志与统计分析

| 功能 | 说明 |
|------|------|
| 指令执行日志 | 时间 / 教师 / 指令原文 / 识别结果 / 执行状态 |
| 识别失败分析 | 高频失败词 → 建议加入词库 |
| 使用频率统计 | 常用指令 TOP10 / 教师使用频次 / 高峰时段 |
| 唤醒率统计 | 唤醒成功/失败次数 |
| 日志保留 | 详细日志 30 天 / 月度汇总长期保留 |
| 筛选 | 按教师/日期/指令类型/执行状态筛选 |

## 3. 桌面端模块设计

### 3.1 唤醒引擎

| 功能 | 说明 |
|------|------|
| 离线唤醒 | 本地引擎持续监听，不依赖网络 |
| 唤醒反馈 | TTS 播报"我在" + 界面图标高亮 |
| 唤醒后超时 | X 秒无有效指令自动回到待唤醒（取管理端配置） |
| 防误唤醒 | 连续 N 次误唤醒后自动降低一档灵敏度 |

### 3.2 语音识别（ASR）

| 功能 | 说明 |
|------|------|
| 唤醒后录音 | 检测到静音 X 秒后自动结束 |
| 云端 ASR | 发送录音到语音引擎识别为文字 |
| 热词优化 | 将管理端词库附加到 ASR 请求中 |
| 多候选 | 返回 Top-N 候选供 NLU 择优匹配 |

### 3.3 指令解析（NLU）

| 功能 | 说明 |
|------|------|
| 意图匹配 | ASR 文本 → 匹配预定义意图 |
| 槽位提取 | 从文本提取参数（如目标名称/数字/颜色） |
| 模糊匹配 | 同义指令归一化（"下一张"="下一页"） |
| 参数灵活分发 | "打开"类参数按类型分发到不同匹配器 |
| 未识别处理 | TTS "没听清，请再说一遍" + 界面提示 |

### 3.4 指令执行器

| 指令类别 | 执行目标 |
|---------|---------|
| 课件控制 | 白板软件翻页/跳转/打开/关闭 |
| 板书控制 | 画笔/橡皮/清屏/保存 |
| 教学工具 | 计时器/聚光灯/放大镜/骰子/小黑板 |
| 系统操作 | 打开外部应用/浏览器/系统功能 |
| 多媒体 | 播放/暂停/快进/快退/音量调节 |
| 课堂互动 | 随机点名/小组评分 |
| 语音搜索 | 搜索资源库 / AI即时问答弹窗 |

### 3.5 TTS 语音反馈

| 场景 | 播报内容 |
|------|---------|
| 唤醒成功 | "我在" |
| 指令执行成功 | （可选）简短确认，如"已翻到第5页" |
| 未识别 | "没听清，请再说一遍" |
| 执行失败 | "操作失败，请重试" |
| 点名播报 | 学生姓名 |
| 超时退回 | （无播报，直接回待唤醒状态） |

### 3.6 状态指示器

| 状态 | 视觉呈现 |
|------|---------|
| 待唤醒 | 麦克风图标（灰色半透明浮窗） |
| 已唤醒 | 图标高亮 + 呼吸光晕 + "请说指令"提示 |
| 识别中 | 音量波形动画 |
| 执行中 | 转圈 / Toast 提示 |
| 未识别 | 图标抖动 + 短暂提示"没听清" |

## 4. 语音交互流程图

```
                   ┌─────────────┐
                   │  待唤醒状态  │
                   └──────┬──────┘
                          │ 用户说 "果仁果仁"
                          ▼
                   ┌─────────────┐
                   │  已唤醒状态  │ TTS: "我在"
                   │  (等待指令)  │ 图标: 高亮呼吸
                   └──────┬──────┘
                    ┌─────┴─────┐
                    │           │
             用户说出指令    8秒无指令
                    │           │
                    ▼           ▼
            ┌────────────┐  ┌──────────┐
            │ ASR 识别   │  │ 回到待唤醒│
            └─────┬──────┘  └──────────┘
                  │ 识别结果
                  ▼
            ┌────────────┐  失败
            │ NLU 指令   ├────────► TTS: "没听清"
            │ 解析       │          回到待唤醒
            └─────┬──────┘
                  │ 匹配成功
                  ▼
            ┌────────────┐
            │ 执行器分发  │
            └─────┬──────┘
                  │
        ┌────────┴────────┐
        │                 │
   执行成功           执行失败
        │                 │
        ▼                 ▼
  TTS: "已翻到"     TTS: "操作失败"
  回到待唤醒          回到待唤醒
```

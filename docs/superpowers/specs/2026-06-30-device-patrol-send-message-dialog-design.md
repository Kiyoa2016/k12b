# 设备巡视 — 发送信息弹窗设计

## 概述

在设备巡视界面的视频巡视弹窗（`PatrolDialog`）中，点击底部远程控制栏的"发送信息"按钮时，弹出一个独立的消息发送弹窗（`SendMessageDialog`），用于向指定教室设备发送广播消息。

## 触发入口

- **位置**：`PatrolDialog` 底部远程控制栏（已有"发送信息"按钮）
- **当前行为**：点击后显示 Snackbar 提示
- **目标行为**：点击后打开 `SendMessageDialog`

## 整体布局

- 组件类型：MUI `Dialog`
- 尺寸：`maxWidth="md"`（约 960px），`fullWidth`
- 布局：左右分栏 + 底部按钮栏

```
┌─────────────────────────────────────────────────┐
│  发送信息 — 东教学楼101教室                    ✕  │
├──────────────────────┬──────────────────────────┤
│                      │  消息内容             150 │
│   🖥️ 教室屏幕预览   │  [文本输入框]             │
│                      │  字体大小  ○小 ○中 ○大   │
│   消息在设备上的     │  字体颜色  ■ ■ ■ ■ ■ ■  │
│   模拟播放效果       │  快捷模板  ▾ 请选择...    │
│                      │  播放方式  ○跑马灯 ○弹窗  │
│                      │  播放次数  [-] 2 [+]     │
│                      │     (仅跑马灯时显示)      │
├──────────────────────┴──────────────────────────┤
│              [取消]    [发送指令]                  │
└─────────────────────────────────────────────────┘
```

## 左栏 — 教室屏幕预览

### 结构
- 模拟教室设备屏幕，16:10 比例，深色渐变背景
- 顶部信息条：教室名称、在线状态
- 中间为消息播放区域
- 底部任务栏：设备编号、时间显示

### 预览联动
- 右侧任何配置变更**立即**反映到左侧预览
- 字体大小 / 颜色实时更新
- 切换播放模式时切换预览动画

### 跑马灯模式预览
- 消息文字从屏幕右侧移入 → 向左水平滚动 → 从左侧移出
- 使用 CSS animation（`marquee`）实现平滑滚动效果

### 全局弹窗模式预览
- 屏幕中央显示弹窗卡片，消息内容居中
- 背景半透明遮罩
- 弹窗卡片带圆角和阴影

## 右栏 — 消息配置区

### 1. 消息内容
- MUI `TextField`，`multiline`，`minRows={3}`，`maxRows={6}`
- 右下角字数统计 `n/150`，超限时变红提示
- 输入框 placeholder："请输入消息内容..."

### 2. 字体大小
- 三个预设值，使用 Chip/Button 选择组
- 小 = 16px，中 = 24px，大 = 36px
- 默认选中"中"

### 3. 字体颜色
- 6 个预设色块（圆圈），点击选中带勾选标记
- 预设色（适合深色屏幕显示）：
  - `#ffffff`（白）
  - `#ff4444`（红）
  - `#ffbb00`（黄）
  - `#44ff44`（绿）
  - `#44bbff`（蓝）
  - `#ff88ff`（粉紫）
- 默认选中白色

### 4. 快捷模板
- MUI `Select` 下拉框，placeholder "选择模板..."
- 带分组（`Menu` 分组渲染）
- 选中后自动填入消息内容框

**预置模板分类及内容：**

| 分类 | 模板内容 |
|------|---------|
| 通用通知 | 请同学们回到座位，准备上课 |
| 通用通知 | 下课时间到，请老师有序组织学生离开教室 |
| 考试/自习 | 考试进行中，请保持考场安静 |
| 考试/自习 | 自习时间，请同学们保持安静，认真学习 |
| 安全/应急 | 消防演练开始，请全体师生有序撤离 |
| 安全/应急 | 紧急情况，请保持冷静，听从指挥 |

### 5. 播放方式
- 两个 Radio 按钮：`跑马灯` / `全局弹窗`
- 默认选中"跑马灯"

### 6. 播放次数
- **仅跑马灯模式**时显示
- 数字输入框 + `[-]` `[+]` 按钮
- 范围 1~99，步长 1，默认值 3
- 全局弹窗模式时隐藏

## 底部按钮

- **取消**：关闭弹窗，不保存
- **发送指令**：校验 → 构造 payload → 关闭弹窗 → 回调通知父组件发送结果

### 校验规则
1. 消息内容不能为空
2. 消息内容 ≤ 150 字
3. 不通过时在输入框下方显示错误提示

## 数据接口

### Payload 结构

```typescript
interface SendMessagePayload {
  classroomId: string;
  content: string;
  fontSize: 'small' | 'medium' | 'large';
  fontColor: string;
  playMode: 'marquee' | 'popup';
  playCount?: number;      // 仅 marquee 时
  templateId?: string;
}
```

### 交互流程

```
PatrolDialog
  └─ 点击"发送信息"
      └─ handleRemoteCommand → 打开 SendMessageDialog
           └─ 用户编辑配置
                └─ 点击"发送指令"
                     └─ 校验通过
                          └─ 构造 payload
                          └─ 关闭 SendMessageDialog
                          └─ 回调 → PatrolDialog 显示 Snackbar
                               └─ "已发送信息至 东教学楼101教室"
```

## 组件架构

```
src/app/components/
  └── DevicePatrol.tsx          (添加导入和新状态)
  └── SendMessageDialog.tsx     (新组件)
       ├── 左栏：MessagePreview
       └── 右栏：MessageConfig
            ├── 消息内容编辑
            ├── 字体设置（大小 + 颜色）
            ├── 快捷模板选择
            ├── 播放方式 + 播放次数
            └── 底部按钮

```

- `SendMessageDialog` 为独立组件，从 `DevicePatrol.tsx` 移出到单独文件
- `PatrolDialog` 中只增加 `sendMessageOpen` 状态和 `handleSendMessage` 回调

## 实现要点

1. **CSS 动画**：跑马灯滚动效果使用 `keyframes` 动画，消息从右侧移入左侧移出
2. **字数限制**：`TextField` 的 `inputProps.maxLength={150}` + 实时字数统计
3. **模板选择**：使用 MUI `Select` 的 `groups` 特性或自定义分组渲染
4. **预览同步**：预览区使用 useMemo 响应右侧状态变化
5. **仅跑马灯显示次数**：条件渲染 `{playMode === 'marquee' && <播放次数控件/>}`

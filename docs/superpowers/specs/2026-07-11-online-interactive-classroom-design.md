# 线上互动课堂 — 设计文档

## 概述

在"校级菜单"下新增"云空堂"分组，其下提供"线上互动课堂"模块。该模块是以直播为核心的互动教学工具，支持实时拍照上传、截屏插入、答题器，以及通过二维码/链接进行公网直播分享，可切换直播画面布局。

## 菜单结构变更

在 `src/app/App.tsx` 的 `menuGroups` 中，于 `school-level`（校级菜单）的 `children` 数组内新增：

```
校级菜单
├── ...（现有项保持不变）
├── 云空堂（新增分组）
│   └── 线上互动课堂（新增子项，pageId: 'online-classroom'）
└── 集控管理（现有）
```

- 云空堂分组图标：`Videocam` 或 `LiveTv`
- 线上互动课堂菜单项使用 `Videocam` 图标

### 功能模块详情

#### 1. 直播画面区（左侧）

- **摄像头画面**：通过 `navigator.mediaDevices.getUserMedia` 获取本地摄像头视频流
- **叠加层**：拍照上传的图片或截屏图片可以作为画中画（PiP）叠加在画面之上
- **布局模式**（四种预设）：
  - *教师全屏*：摄像头画面全屏显示
  - *课件全屏*：上传的图片/课件占满整个画面
  - *画中画*：主画面为课件，右下角小窗显示教师摄像头
  - *三分屏*：左半教师、右上半课件、右下半互动区（答题统计）

#### 2. 右侧工具栏

**拍照上传**：
- 「拍照」按钮：调用 `getUserMedia` 打开摄像头，显示取景器，点击拍照定格画面
- 「上传」按钮：打开文件选择器，支持选择图片文件
- 拍摄/选择的图片预览缩略图，点击可插入到直播画面
- 已插入的图片列表，支持删除

**截屏插入**：
- 一键截取当前直播画面的 canvas 快照
- 截屏图片自动添加到媒体列表
- 可插入到画面叠加层或保存

**答题器**：
- 教师创建单选题：输入题目文字 + 最多4个选项
- 发送题目后，模拟学生端作答（模拟随机派发答案）
- 实时统计并显示各选项的投票结果（横向柱状图）
- 支持结束答题、重置答题

**直播分享**：
- 生成一个模拟的课堂链接（如 `https://live.example.com/classroom/{id}`）
- 显示二维码（使用 `qrcode` 库生成）
- 「复制链接」按钮复制到剪贴板
- 二维码弹窗展示，供手机扫码

#### 3. 底部控制栏

- 布局模式切换按钮组（四种布局图标按钮）
- 直播状态指示器（`● 直播中` / `○ 已停止`）
- 「开始/停止推流」按钮
- 「分享」按钮（快捷打开分享弹窗）

## 状态管理

所有状态在 `OnlineInteractiveClassroom` 组件内部使用 `useState` 管理：

| 状态 | 类型 | 说明 |
|------|------|------|
| `isLive` | boolean | 直播推流状态 |
| `layoutMode` | 'teacher' \| 'slide' \| 'pip' \| 'three-panel' | 当前布局模式 |
| `cameraStream` | MediaStream \| null | 摄像头视频流 |
| `mediaItems` | MediaItem[] | 拍照/上传/截屏的图片列表 |
| `activeOverlay` | string \| null | 当前叠加在画面上的图片ID |
| `quizQuestion` | string | 当前题目文本 |
| `quizOptions` | string[] | 选项列表 |
| `quizVotes` | Record<string, number> | 各选项得票数 |
| `quizActive` | boolean | 答题是否进行中 |
| `shareDialogOpen` | boolean | 分享弹窗开关 |
| `cameraDialogOpen` | boolean | 拍照弹窗开关 |

## UI 组件使用

- 使用 MUI 的 `Box`, `Typography`, `Button`, `IconButton`, `Chip`, `Dialog`, `Grid`, `Card` 等现有组件
- 使用 Tailwind CSS 进行样式（沿用项目现有模式）
- 使用 `@mui/icons-material` 中的图标

## 项目集成

在 `App.tsx` 中需要修改：

1. **导入** `OnlineInteractiveClassroom`
2. **pageId 类型联合**：添加 `'online-classroom'` 到 `currentPage` state 的类型
3. **菜单配置**：在 `school-level` 下添加 `cloudhall-parent` 分组
4. **路由渲染**：在页面渲染条件链中添加 `'online-classroom'` 分支

## 依赖

- `qrcode` npm 包：用于生成分享二维码（`npm install qrcode` + `@types/qrcode`）
  - 使用 `QRCode.toCanvas()` 或 `QRCode.toDataURL()` 生成二维码嵌入页面

## 错误处理

- 摄像头权限被拒绝时显示友好的提示信息
- 图片上传格式验证（仅支持常见图片格式）
- 答题器输入验证（题目和至少2个选项）
- 空状态展示（无媒体项、无答题记录等）

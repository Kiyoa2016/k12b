# 学生端互动课堂 — 设计文档

## 概述

为互动课堂提供学生端体验。学生通过扫描二维码或点击链接进入课堂，输入姓名后即可观看老师直播画面并参与答题互动。

## 技术约束

- 全平台适配（手机、平板、电脑），响应式设计
- 使用 MUI + Tailwind CSS，与老师端技术栈一致
- 所有数据为前端模拟（暂无后端）
- 学生身份通过"输入姓名"识别，无需账号系统

## 页面流转

```
扫码/点链接 → 加入课堂页(输入姓名) → 课堂观看页(全屏内容+浮动UI) → 答题弹窗 → 结果展示
```

## 页面一：加入课堂页面

### 布局

居中卡片式设计，自上而下排列：

```
┌──────────────────────────────────────┐
│  ← 返回                              │
│         ┌────────────────┐           │
│         │   📹            │          │
│         │  互动课堂       │          │
│         │  函数与极限     │          │
│         │  张老师 · 数学  │          │
│         │                │          │
│         │  请输入姓名     │          │
│         │  [____________] │          │
│         │  [ 加入课堂 ]   │          │
│         └────────────────┘           │
└──────────────────────────────────────┘
```

### 交互规则

- 页面加载时从 URL 参数获取 classroomId，显示课堂信息
- 姓名输入框非空时「加入课堂」按钮才可点击
- 点击加入后跳转到课堂观看页面

### 状态处理

- 课堂不存在或已结束 → 显示提示 "该课堂不存在或已结束"
- 网络错误 → 显示 "加载失败，请检查网络"

## 页面二：课堂观看页面

### 布局

全屏直播内容 + 浮动 UI 元素：

```
┌──────────────────────────────────────┐
│  ← 退出课堂          👥 在线 3人  👤  │  ← 半透明顶部栏
│──────────────────────────────────────│
│          老师端直播内容（全屏）        │
│          - 板书（电脑桌面.png）        │
│          - 教师摄像头画面              │
│          - 画中画模式                  │
│          （跟随老师端布局自动同步）     │
│                                      │
│                                      │
│──────────────────────────────────────│
│  💬 老师发起了一个投票   [立即参与]    │  ← 底部通知条
└──────────────────────────────────────┘
```

### 交互规则

- 顶部栏半透明浮动，点击退出返回加入页
- 右上角 `👥` 展开成员列表浮层
- 内容区域全屏显示老师当前布局内容（板书/摄像头/画中画）
- 老师发起答题时，底部滑入通知条
- 老师切换布局时，学生端自动同步

### 状态处理

- 老师未开始直播 → 显示 "等待老师开始上课..."
- 连接断开 → 显示 "连接已断开，正在重连..."
- 成员列表为空 → 显示 "暂无其他成员"

## 页面三：答题交互

三个阶段：

### 阶段一：通知

老师发起答题 → 学生端底部滑入通知条：
- 文字 "老师发起了一个投票"
- 按钮 "[立即参与]"
- 不影响当前观看内容

### 阶段二：答题弹窗

点击"立即参与"后弹出居中 Dialog：

- 标题区：❓ 答题器 + 关闭按钮
- 题目文字
- 选项列表（单选，圆形 Radio）
- 「提交答案」按钮（底部）
- 未选择选项时按钮禁用

### 阶段三：结果展示

提交后弹窗自动切换到结果视图：

- 标题区：📊 答题结果
- 题目文字
- 你的答案 + ✅/❌ 标识
- 各选项横向柱状图（票数 + 百分比）
- 共 N 人参与
- 「关闭」按钮

### 交互规则

- 提交后弹窗不自动关闭，等待学生手动关闭
- 关闭弹窗后继续观看
- 老师结束答题 → 所有学生自动显示最终统计结果
- 老师发起新答题 → 旧结果关闭，新通知弹出
- 未参与的学生结果页显示 "你未参与本次投票"

## 模拟数据接口

所有数据目前为前端模拟：

```
classroom = { id, name, teacher, subject }
participants = [{ id, name, role, online }]
quiz = { question, options, votes, correctAnswer }
```

老师端和学生端通过共享的模拟数据状态进行通信（后续接入真实 WebSocket）。
